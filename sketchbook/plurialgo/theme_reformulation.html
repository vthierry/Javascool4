Le mécanisme de reformulation

Introduction

Ce mécanisme très utile permet de réécrire le programme courant de l'éditeur (donc attention si plusieurs programmes sont ouverts) en introduisant des sous-programmes :

  1. si aucune portion de code n'est sélectionnée, c'est tout le programme principal qui est reformulé quand on clique sur le bouton Reformuler.
  2. si une portion de code est sélectionnée, c'est elle qui est transformée.

Reformulation sans sélection de code

Un exemple a été donné dans la section fonctions.

Il est possible de changer l'option de calcul en "procedure", mais le résultat final n'est pertinent que si le langage final autorise le passage de paramètres par référence (Visual Basic, Larp).

La même réserve prévaut pour l'option de saisie "procedure". Mais si vous programmez en Javascool ou en Python, vous pouvez vous consoler avec l'option de saisie "formulaire" !

Reformulation avec sélection de code

Si l'on veut introduire une fonction calculant le taux de remise (et non le prix total comme précédemment), le code à reformuler doit être sélectionné :

  1. Téléchargez le programme initial (exemple Javascool, Python, Xcas ou Visual Basic).
  2. Complétez l'onglet "Principal" (exemple), en indiquant également le langage du programme initial.
  3. Cliquez sur le bouton Reformuler.

Les options "saisie", "affichage" et "regroupement" de l'onglet Principal sont ignorées : inutile de les changer par conséquent.
