Le mécanisme de transformation 1-n

Introduction

Ce mécanisme très utile encapsule dans une boucle (Pour ou Tantque) le programme courant de l'éditeur (donc attention si plusieurs programmes sont ouverts) :

  1. si aucune portion de code n'est sélectionnée, c'est tout le programme principal qui est transformé quand on clique sur le bouton Transformer 1-n.
  2. si une portion de code est sélectionnée, c'est elle qui est transformée.

Dans le premier cas, la transformation ne peut s'appliquer qu'à un programme que PluriAlgo sait analyser : ce programme doit donc être exprimé dans un langage pédagogique (Javascool, Algobox, Larp, Python) ou en Visual Basic.

Dans le second cas, il n'y a pas de restriction sur le langage, mais le résultat obtenu est moins précis : par exemple, les variables introduites dans l'onglet Boucles ne seront pas déclarées. De plus, l'option "vectorisation" sera ignorée.

Transformation sans sélection de code

Vous allez ici appliquer la transformation à un programme lisant une note, exprimé dans un langage pédagogique à fixer dans l'onglet Principal : pour l'obtenir, complétez l'onglet (télécharger), puis cliquez sur le bouton Nouveau.

Exemple 1 : option "somme"

Complétez l'onglet Boucles (télécharger), afin de calculer la somme des notes saisies.

Exemple 2 : option "comptage"

Complétez l'onglet Boucles (télécharger), afin de compter le nombre d'étudiants admis.

Exemple 3 : options "minimum" et "maximum"

Complétez l'onglet Boucles (télécharger), afin de trouver la note minimale et la note maximale.

Exemple 4 : option "recherche"

Complétez l'onglet Boucles, afin de répéter la saisie jusqu'à obtenir une note valide (exemple 1, exemple 2), ou tant que la note est valide (exemple 3).

Exemple 5 : option "vectorisation"

Reprenez un des exemples précédents en complétant le champ "vectorisation" (avec la valeur note), afin de transformer la variable réelle "note" en tableau de réels.
