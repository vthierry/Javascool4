Le mécanisme de transformation 1-n

Introduction

Ce mécanisme très utile encapsule dans une boucle (Pour ou Tantque) le programme courant de l'éditeur (donc attention si plusieurs programmes sont ouverts) :

  1. si aucune portion de code n'est sélectionnée, c'est tout le programme principal qui est transformé quand on clique sur le bouton Transformer 1-n.
  2. si une portion de code est sélectionnée, c'est elle qui est transformée.

Dans le premier cas, la transformation ne peut s'appliquer qu'à un programme que PluriAlgo sait analyser : ce programme doit donc être exprimé en Javascool, Algobox, Larp, Python, Xcas ou Visual Basic.

Dans le second cas, il n'y a pas de restriction sur le langage, mais sur l'option "vectorisation" qui sera ignorée. De plus, les variables introduites dans l'onglet Boucles ne seront pas déclarées.

Exemples

Vous êtes invité à transformer un programme Javascool lisant une note (télécharger), en sélectionnant ou non l'instruction de saisie. Les tests peuvent être effectués dans un autre langage en traduisant ce programme.

Exemple 1 : option "somme"

Complétez l'onglet Boucles (télécharger), afin de calculer la somme des notes saisies (grâce au bouton Transformer 1-n).

Exemple 2 : option "comptage"

Complétez l'onglet Boucles (télécharger), afin de compter le nombre d'étudiants admis.

Exemple 3 : options "minimum" et "maximum"

Complétez l'onglet Boucles (télécharger), afin de trouver la note minimale et la note maximale.

Exemple 4 : option "recherche"

Complétez l'onglet Boucles, afin de répéter la saisie jusqu'à obtenir une note valide (version 1 ou version 2).

Exemple 5 : option "vectorisation" (sans sélection de code)

Reprenez l'exemple 1 en complétant le champ "vectorisation" avec la valeur note, afin de transformer cette variable en tableau à une dimension.
