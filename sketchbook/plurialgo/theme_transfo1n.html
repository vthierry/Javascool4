Le mécanisme de transformation 1-n

Introduction

Ce mécanisme encapsule dans une boucle (Pour ou Tantque) le programme courant de l'éditeur (donc attention si plusieurs programmes sont ouverts) :

  1. si aucune portion de code n'est sélectionnée, c'est tout le programme principal qui est répété quand on clique sur le bouton Transformer 1-n.
  2. si une portion de code est sélectionnée, c'est uniquement elle qui est répétée.

Dans le premier cas, la transformation ne peut s'appliquer qu'à un programme que PluriAlgo sait analyser : ce programme doit donc être exprimé en Javascool, Algobox, Larp, Python, Xcas ou Visual Basic.

Dans le second cas, il n'y a pas de restriction sur le langage, mais les variables introduites dans l'onglet Boucles ne seront pas déclarées et l'option "vectorisation" sera ignorée.

Exemples

Vous êtes invité à passer d'un programme lisant une note (avec ou sans code sélectionné) à un programme lisant plusieurs notes :

  • exemple 1 (télécharger) : somme des notes.
  • exemple 2 (télécharger) : comptage des notes >=10.
  • exemple 3 (télécharger) : note minimale et note maximale.
  • exemple 4 (version 1 ou version 2) : recherche d'une note valide (entre 0 et 20).

Après avoir complété l'onglet "Boucles", il vous reste à cliquer sur le bouton Transformer 1-n.
